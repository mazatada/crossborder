# 実装計画書 — 0. 基盤・共通

対象: サービス起動／構成管理、認証基盤、共通エラー、ヘルス/バージョン、ロギング&トレーシング、ルーティング可視化、（将来）レート制限・サイズ上限

---

## 1. 目的と範囲
- **目的**: backend を「安全に起動し、安定運用できる最小単位」に仕上げる。
- **範囲**: Flask 初期化、CORS、API Key 認証（免除ルート含む）、統一エラーJSON、/v1/health と /v1/version、構造化ログ＋trace_id、ルート可視化、（任意）レート制限とリクエストサイズ制御。
- **非範囲**: 業務APIの中身、DBスキーマ拡張、非同期ジョブ実行（別章）。

---

## 2. 成果物
- 起動可能な Docker Compose 一式（backend+db）
- `app/factory.py`, `app/auth.py`, `app/errors.py` の完成版
- `/v1/health`, `/v1/version` 実装
- 構造化ログ（JSON）と `trace_id` 付与
- 運用 Runbook（起動・設定・確認コマンド）
- 自動テスト（最小: health/version/auth/エラーの4系）
