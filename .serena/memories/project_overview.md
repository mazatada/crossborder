# Cross-border EC automation stack
- Purpose: this repo implements the regulatory core for a cross-border e-commerce automation suite—translation/ingredient normalization, HS classification, clearance-doc generation, FDA Prior Notice, audit trail, webhook/inbound integration, and a job queue that surfaces long-running tasks for document generation/PN submission. CMS/marketing functions are explicitly out of scope as documented in `docs/越境EC成分表翻訳アプリ仕様書_v1_0.md` and `docs/spec/SPEC.md`.
- Architecture: Docker Compose wires up a Postgres `db`, Flask `backend`, and additional `worker`/`scheduler` services that run the same backend image with `app.jobs.cli`. The backend is a Flask app created in `backend/app/factory.py` that registers multiple v1 blueprints, uses a custom `app/db.py` shim on SQLAlchemy 2.x, and records structured audit events before/after job runs. `backend/app/jobs/cli.py` provides the scheduler/worker loops, handlers live under `backend/app/jobs/handlers`, and `app/webhook.py` posts JSON events to `WEBHOOK_URL` with HMAC signatures.
- Frontend: `frontend/` is a Vite+React TypeScript project (`npm run dev`) that queries `import.meta.env.VITE_API_BASE` for `/v1` endpoints, renders minimal MVP screens (HS/Pack/PN) using Material UI components (`Stack`, `Tabs`, `Paper`, etc.), and exposes helpers in `frontend/src/api.ts` and `frontend/src/types.ts`.
- Supporting artifacts: `openapi.yaml` defines the API contract, `docs/ui/frontend_mvp_ui_spec.md` lists the required screens, `docs/implementation_plan_*.md` capture roadmap for plans like devex/quality, security, audit, etc., and `schemas/` holds JSON Schema for key payloads.
- Key entrypoints: `docker-compose up --build` (or `docker compose up --build`), `backend/entrypoint.sh` launching `flask run`, `frontend/npm run dev`, `backend/cli.py` with `--mode worker|scheduler`, `backend/requirements.txt` for Python deps, `frontend/package.json` for Node deps, `backend/alembic` + `migrations` for schema.
